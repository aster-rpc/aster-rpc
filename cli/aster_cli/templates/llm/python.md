## What is Aster?

Aster is a peer-to-peer RPC framework built on [Iroh](https://github.com/n0-computer/iroh) QUIC networking. It provides:

- **Type-safe RPC** with four streaming patterns (unary, server-stream, client-stream, bidi-stream)
- **Cross-language serialization** via Fory (XLANG, NATIVE, ROW modes)
- **Content-addressed blobs** (BLAKE3-verified storage and transfer)
- **CRDT documents** (replicated key-value store with live sync)
- **Gossip pub-sub** (topic-based broadcast)
- **Trust model** with enrollment credentials and capability-based auth

Install: `pip install aster-python`

## Defining Data Types

All request/response types MUST be Python `@dataclass` classes. All fields MUST have default values.

### `@wire_type(tag)` — Stable Wire Identity

```python
from dataclasses import dataclass
from aster import wire_type

@wire_type("billing/Invoice")
@dataclass
class Invoice:
    amount: float = 0.0
    customer_id: str = ""
    line_items: list[str] = None  # use None default for collections

    def __post_init__(self):
        if self.line_items is None:
            self.line_items = []
```

**Tag format**: `namespace/TypeName` (split on last `/`).

**Rules**:
- Tags MUST be stable across versions — they are the wire identity
- Nested dataclasses must also be tagged
- Omitting `@wire_type` auto-derives a tag from `module.qualname` (e.g. `myapp.models.Invoice`). This is deterministic and stable as long as you don't move or rename the class. Use explicit `@wire_type` when the wire identity must survive refactors.
- Enums used in dataclass fields are auto-tagged the same way. `@wire_type` can be applied to enums for explicit control.

### Supported field types

`int`, `float`, `str`, `bool`, `bytes`, `Enum`, `IntEnum`, `list[T]`, `dict[K, V]`, `Optional[T]`, nested `@dataclass`, `set[T]`

## Defining Services

### `@service` — Mark a Class as an RPC Service

```python
from aster import service, rpc, server_stream, client_stream, bidi_stream

@service(name="MathService", version=1)
class MathService:
    """A simple math service."""

    @rpc
    async def add(self, req: AddRequest) -> AddResponse:
        return AddResponse(result=req.a + req.b)
```

**`@service` parameters**:
- `name` (str): Service name (default: class name)
- `version` (int): Version number (default: 1)
- `serialization` (SerializationMode | list): `XLANG` (default), `NATIVE`, or `ROW`
- `scoped` (str): `"shared"` (default, singleton) or `"stream"` (one instance per client connection)

### RPC Patterns

#### Unary — `@rpc`
Single request, single response.

```python
@rpc
async def echo(self, req: EchoRequest) -> EchoResponse:
    return EchoResponse(message=req.message)
```

#### Server Streaming — `@server_stream`
Single request, multiple responses. Method MUST be an async generator.

```python
from typing import AsyncIterator

@server_stream
async def list_items(self, req: ListRequest) -> AsyncIterator[Item]:
    for item in self.items:
        yield Item(name=item.name)
```

#### Client Streaming — `@client_stream`
Multiple requests, single response. Parameter MUST be `AsyncIterator[T]`.

```python
@client_stream
async def upload(self, chunks: AsyncIterator[Chunk]) -> UploadResult:
    total = 0
    async for chunk in chunks:
        total += len(chunk.data)
    return UploadResult(bytes_received=total)
```

#### Bidirectional Streaming — `@bidi_stream`
Multiple requests and responses. Async generator that receives and yields.

```python
@bidi_stream
async def chat(self, messages: AsyncIterator[Message]) -> AsyncIterator[Message]:
    async for msg in messages:
        yield Message(text=f"Echo: {msg.text}")
```

### Method Options

All RPC decorators accept:
- `timeout` (float): Timeout in seconds
- `idempotent` (bool): Safe to retry on failure (default: False)
- `serialization` (SerializationMode): Override per-method

```python
@rpc(timeout=30.0, idempotent=True)
async def process(self, req: Request) -> Response: ...
```

### Session-Scoped Services

Use `scoped="stream"` for per-connection state:

```python
@service(name="SessionService", version=1, scoped="stream")
class SessionService:
    def __init__(self, peer=None):
        self.peer = peer  # endpoint ID of the connected client
        self.state = {}

    def on_session_close(self):
        """Called when the client disconnects."""
        pass

    @rpc
    async def set_value(self, req: SetRequest) -> SetResponse:
        self.state[req.key] = req.value
        return SetResponse(ok=True)
```

## Error Handling

Raise `RpcError` with a status code to signal failures:

```python
from aster import RpcError, StatusCode

@rpc
async def get_user(self, req: GetUserRequest) -> User:
    user = self.db.get(req.id)
    if not user:
        raise RpcError(
            code=StatusCode.NOT_FOUND,
            message=f"User {req.id} not found",
        )
    return user
```

Common status codes: `OK`, `CANCELLED`, `INVALID_ARGUMENT`, `NOT_FOUND`, `ALREADY_EXISTS`, `PERMISSION_DENIED`, `RESOURCE_EXHAUSTED`, `UNIMPLEMENTED`, `INTERNAL`, `UNAVAILABLE`, `UNAUTHENTICATED`, `DEADLINE_EXCEEDED`.

## Data Directory (Persistence)

Aster uses persistent storage by default so that node identity, blobs, and
documents survive restarts. Configure the data directory via:

1. **Environment variable** (recommended): `export ASTER_STORAGE_PATH=.aster-data`
2. **TOML config** (`aster.toml`): `[storage]` → `path = ".aster-data"`
3. **Inline**: `AsterConfig(storage_path=".aster-data")`

If no storage path is set, an ephemeral in-memory store is used (data lost on exit).

For local network discovery (nodes on the same LAN find each other without relay):
`AsterConfig(local_discovery=True)` or `export ASTER_LOCAL_DISCOVERY=true`

## Running a Server (Producer)

```python
from aster import AsterServer, AsterConfig

async def main():
    config = AsterConfig(storage_path=".aster-data")
    server = AsterServer(
        services=[MathService()],
        peer="math-producer",
        config=config,
    )
    async with server:
        print(f"Listening on {server.endpoint_id}")
        await server.serve()
```

Or set `ASTER_STORAGE_PATH` in the environment and omit the config — it is
picked up automatically.

For production with enrollment credentials:

```python
server = AsterServer(
    services=[MathService()],
    identity=".aster-identity",  # from `aster enroll node`
)
```

## Calling a Service (Consumer)

```python
from aster import AsterClient, AsterConfig

async def main():
    config = AsterConfig(storage_path=".aster-data")
    client = AsterClient(peer="math-consumer", config=config)
    async with client:
        math = await client.client(MathService)
        result = await math.add(AddRequest(a=3, b=5))
        print(result.result)  # 8
```

As with the server, `ASTER_STORAGE_PATH` is picked up automatically if set.


## Iroh Primitives — Blobs

Content-addressed blob storage with BLAKE3 hashing.

```python
from aster import IrohNode, blobs_client

node = await IrohNode.persistent(".aster-data")  # or IrohNode.memory() for ephemeral
blobs = blobs_client(node)

# Store and retrieve
hash_hex = await blobs.add_bytes(b"hello world")
data = await blobs.read_to_bytes(hash_hex)

# Check status
exists = await blobs.blob_has(hash_hex)
status = await blobs.blob_status(hash_hex)  # .status: "complete"|"partial"|"not_found"

# Named tags (protect blobs from garbage collection)
await blobs.tag_set("my-data", hash_hex, "raw")
info = await blobs.tag_get("my-data")        # .name, .hash, .format
await blobs.tag_delete("my-data")

# Share via tickets
ticket = blobs.create_ticket(hash_hex)
data = await blobs.download_blob(ticket)
```

**Key concepts**:
- Blobs are immutable and content-addressed (BLAKE3 hash = identity)
- Untagged blobs are garbage-collected. Always hold a tag for data you want to keep.
- `"hash_seq"` format is for collections (a blob whose content is a sequence of child hashes)

## Iroh Primitives — Documents (CRDT)

Replicated key-value store with conflict-free merging.

```python
from aster import IrohNode, docs_client

node = await IrohNode.persistent(".aster-data")
docs = docs_client(node)

# Create a document and author
doc = await docs.create()
author = await docs.create_author()

# Write and read
await doc.set_bytes(author, b"config/port", b"8080")
value = await doc.get_exact(author, b"config/port")  # b"8080"

# Query by prefix
entries = await doc.query_key_prefix(b"config/")
for entry in entries:
    content = await doc.read_entry_content(entry.content_hash)

# Share and sync
ticket = await doc.share("write")  # or "read"
doc2 = await other_docs.join(ticket)

# Live events
receiver = await doc.subscribe()
while True:
    event = await receiver.recv()
    if event is None:
        break
    if event.kind == "insert_remote":
        print(f"Peer {event.from_peer} wrote {event.entry.key}")
    elif event.kind == "content_ready":
        print(f"Blob {event.hash} now available locally")

# Download policy (selective sync)
await doc.set_download_policy("nothing_except", [b"important/"])
```

**Key concepts**:
- Documents store metadata (key, hash, size, timestamp). Actual content lives in blobs.
- Each entry is keyed by `(author, key)` — multiple authors can write the same key independently.
- Timestamps are wall-clock microseconds. Clock skew between nodes affects ordering.
- Write tickets contain the namespace secret — treat as sensitive.
- `start_sync(peers)` begins active syncing. `leave()` stops it.

## Iroh Primitives — Gossip (Pub/Sub)

Ephemeral topic-based broadcast. No persistence.

```python
from aster import IrohNode, gossip_client
import os

node = await IrohNode.persistent(".aster-data")
gossip = gossip_client(node)

# Subscribe to a topic (32-byte identifier)
topic_id = os.urandom(32)  # or a well-known topic
topic = await gossip.subscribe(topic_id, [peer_id])

# Send
await topic.broadcast(b"hello peers")

# Receive
while True:
    event_type, data = await topic.recv()
    if event_type == "received":
        print(f"Got: {data}")
    elif event_type == "neighbor_up":
        print("Peer joined")
    elif event_type == "neighbor_down":
        print("Peer left")
    elif event_type == "lagged":
        # MUST recreate subscription — this is terminal
        break
```

**Key concepts**:
- Topic ID is rendezvous, NOT authorization. Anyone who knows it can join.
- `delivered_from` is the last-hop peer, NOT the original author. Sign payloads yourself if authorship matters.
- `lagged` is terminal — the subscription is closed and must be recreated.
- Messages are ephemeral (in-memory only). Use docs or blobs for persistence.

## Iroh Primitives — QUIC Networking

Low-level QUIC streams and datagrams for custom protocols.

```python
from aster import create_endpoint, create_endpoint_with_config, EndpointConfig

# Create an endpoint
config = EndpointConfig(alpns=[b"my-protocol/1"])
ep = await create_endpoint_with_config(config)

# Connect to a peer
conn = await ep.connect(peer_id, b"my-protocol/1")

# Bidirectional streams
send, recv = await conn.open_bi()
await send.write_all(b"request data")
await send.finish()
response = await recv.read_to_end(65536)

# Accept connections (server side)
conn = await ep.accept()
send, recv = await conn.accept_bi()
data = await recv.read_to_end(65536)
await send.write_all(b"response")
await send.finish()

# Unreliable datagrams (low latency, no ordering)
conn.send_datagram(b"ping")
data = await conn.read_datagram()

# Connection info
print(conn.remote_id())
print(f"RTT: {conn.rtt_ms():.1f}ms")
print(f"MTU: {conn.current_mtu()}")
```

**EndpointConfig options**:
- `alpns`: Protocol identifiers (required)
- `secret_key` (bytes): Use a specific 32-byte ed25519 key for stable identity
- `relay_mode` (str): Relay server strategy
- `bind_addr` (str): e.g. `"0.0.0.0:9000"`
- `enable_monitoring` (bool): Enable `remote_info()` tracking
- `enable_local_discovery` (bool): Enable mDNS LAN discovery (nodes find each other without relay)
- `clear_ip_transports` / `clear_relay_transports`: Force relay-only or direct-only

## Complete Example — Service + Producer + Consumer

```python
from dataclasses import dataclass
from typing import AsyncIterator
from aster import (
    service, rpc, server_stream, wire_type,
    AsterServer, AsterClient, AsterConfig, RpcError, StatusCode,
)

# ── Types ──

@wire_type("todo/Task")
@dataclass
class Task:
    id: str = ""
    title: str = ""
    done: bool = False

@wire_type("todo/AddTaskRequest")
@dataclass
class AddTaskRequest:
    title: str = ""

@wire_type("todo/GetTaskRequest")
@dataclass
class GetTaskRequest:
    id: str = ""

@wire_type("todo/ListRequest")
@dataclass
class ListRequest:
    pass

# ── Service ──

@service(name="TodoService", version=1)
class TodoService:
    def __init__(self):
        self.tasks: dict[str, Task] = {}
        self.counter = 0

    @rpc
    async def add_task(self, req: AddTaskRequest) -> Task:
        self.counter += 1
        task = Task(id=str(self.counter), title=req.title)
        self.tasks[task.id] = task
        return task

    @rpc
    async def get_task(self, req: GetTaskRequest) -> Task:
        task = self.tasks.get(req.id)
        if not task:
            raise RpcError(code=StatusCode.NOT_FOUND, message=f"Task {req.id} not found")
        return task

    @server_stream
    async def list_tasks(self, req: ListRequest) -> AsyncIterator[Task]:
        for task in self.tasks.values():
            yield task

# ── Producer ──

async def run_producer():
    config = AsterConfig(storage_path=".aster-data/producer")
    async with AsterServer(services=[TodoService()], peer="todo-producer", config=config) as server:
        print(f"Producer: {server.endpoint_id}")
        await server.serve()

# ── Consumer ──

async def run_consumer(producer_addr):
    config = AsterConfig(storage_path=".aster-data/consumer")
    async with AsterClient(peer="todo-consumer", config=config) as client:
        todo = await client.client(TodoService)

        task = await todo.add_task(AddTaskRequest(title="Write docs"))
        print(f"Created: {task.title} (id={task.id})")

        async for t in todo.list_tasks(ListRequest()):
            print(f"  - [{t.id}] {t.title}")
```

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Dataclass fields without defaults | Every field needs a default: `name: str = ""` |
| Missing `@wire_type` in XLANG mode | Add `@wire_type("namespace/Type")` to all dataclasses |
| `@server_stream` method uses `return` instead of `yield` | Must be an async generator — use `yield` |
| `@client_stream` parameter not typed as `AsyncIterator` | Annotate as `AsyncIterator[RequestType]` |
| Forgetting `await` on RPC calls | All RPC methods are async — always `await` |
| Using mutable default (`list[str] = []`) | Use `None` default + `__post_init__` |
| Assuming gossip `delivered_from` = author | Sign payloads at the application layer |
| Not handling gossip `lagged` event | Recreate the subscription when lagged |
| Deleting all blob tags | Untagged blobs get garbage-collected |
| Sharing a write-mode doc ticket publicly | Write tickets contain the namespace secret key |
